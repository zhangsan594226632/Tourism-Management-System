## 一、项目概述

### 1\.1 项目开发背景

随着国内文旅行业快速发展，线下旅行社、纸质景点宣传手册模式存在明显短板：景点信息更新滞后、游客无法线上浏览景点图文、旅游攻略分享渠道缺失、门票预订流程繁琐、景点评价难以集中查看，同时景区管理员无法统一线上维护景点、资讯、用户数据。
为解决文旅信息数字化管理需求，本系统采用**SpringBoot\+Vue 前后端分离架构**搭建旅游管理平台，分为**游客前台端**、**管理员后台端**两大独立业务模块。游客可在线浏览景点、查看旅游资讯、发布游玩攻略、收藏心仪景点、在线购票；后台管理员统一维护景点分类、景点资料、旅游资讯、用户账号、攻略内容，实现文旅信息线上展示、线上管理一体化，降低景区宣传运营成本，提升游客线上游玩服务体验。

### 1\.2 整体开发技术栈

#### 后端技术（SpringBoot）

1. 核心框架：SpringBoot 2\.7，标准 MVC 分层架构（Controller/Service/Mapper/Entity），统一 RESTful 接口规范

2. 持久层框架：MyBatis\-Plus，快速实现分页查询、多条件模糊检索、批量增删改查

3. 数据库：MySQL 8\.0，存储用户、景点、景点类型、旅游攻略、资讯、收藏、购票订单、轮播图全部业务数据

4. 登录鉴权：JWT 无状态 Token 认证，区分游客普通账号、管理员账号双角色，拦截未登录、越权访问接口

5. 文件上传模块：本地文件存储，支持景点实拍图、资讯封面、攻略配图、首页轮播图片上传预览

6. 安全机制：用户登录密码 MD5 不可逆加密存储，表单前后端双重参数校验，Token 设置固定过期时效

7. 工具依赖：Hutool 工具包，封装日期处理、加密工具、分页工具、文件处理工具

#### 前端技术（Vue3 \+ Element Plus）

1. 核心框架：Vue3 \+ VueRouter 路由管理 \+ Pinia 全局状态存储

2. UI 组件库：轻量化简约游客前台页面；Element Plus 组件库搭建管理员后台管理页面

3. 网络请求：Axios 全局请求 / 响应拦截器，自动携带登录 Token、捕获登录失效、统一弹窗提示异常

4. 通用封装组件：首页轮播组件、景点卡片列表、图片上传组件、富文本编辑器（攻略编辑）、数据分页、搜索框、弹窗详情

5. 拓展预留：预留 ECharts 可视化接口，可拓展景点访问量、门票销量统计图表

### 1\.3 系统两类角色划分

1. **普通游客（前台用户）**
账号注册登录；浏览首页轮播、推荐景点、全部景点列表；查看景点详情、游客评论；在线购买景点门票；发布、编辑个人旅游攻略；收藏心仪景点；浏览旅游资讯；个人中心修改密码、管理我的收藏、我的攻略。

2. **系统管理员（后台）**
全平台最高管理权限：管理全部游客账号、后台管理员账号；维护景点类型分类、新增 / 编辑 / 删除景点信息；管理所有用户发布的旅游攻略；发布、编辑、删除旅游资讯；配置前台首页轮播图；查看所有门票订单、景点游客评论，统一管控平台全部文旅数据。

### 1\.4 完整业务流程梳理

1. 登录注册流程：游客前台提供账号注册入口，填写账号密码完成注册；前台、后台独立两套登录页面，登录成功签发 JWT Token，未登录状态无法进入个人中心、购票、发布攻略等功能页面。

2. 游客前台游玩业务流程：首页浏览推荐景点→查看景点详情、评价→线上购买门票；浏览行业旅游资讯；发布个人游玩攻略，收藏喜欢的景点；个人中心统一管理收藏、攻略、修改登录密码。

3. 管理员后台运维流程：维护景点分类、录入景点图文、门票价格、开放时间；审核 / 管理用户发布的旅游攻略；发布景区通知、文旅资讯；配置首页宣传轮播；管理平台全部游客账号，查看所有购票订单与景点评论。

### 2\.1 登录注册模块

#### 2\.1\.1 游客前台登录页面

简约商务风登录表单，包含账号、密码输入框、记住密码勾选框、橙色登录按钮；底部提供「注册用户」跳转入口，未登录访问个人中心、购票功能弹窗拦截。

#### 2\.1\.2 管理员后台登录页面

独立后台登录界面，账号密码校验通过后进入全功能管理后台，拥有平台全部数据操作权限。

#### 2\.1\.3 游客注册页面

必填字段：登录账号、登录密码、确认密码；前端校验密码一致性，后端校验账号唯一性，注册信息存入用户数据表。

### 2\.2 游客前台功能模块（顶部导航栏：首页、景点信息、旅游资讯、个人中心）

#### 2.2.1.系统首页

1. 顶部全屏宣传轮播图，展示景区风景海报；

2. 「景点信息推荐」板块：卡片式展示热门景点预览图、景点名称、收藏 / 访问数量；提供「全部」跳转按钮查看完整景点；

3. 「景点信息展示」板块：分页展示全平台所有景点缩略卡片；

4. 底部「旅游资讯」板块：展示文旅新闻、游玩攻略资讯卡片。

#### 2.2.2 景点信息模块

1. 全部景点列表页面：分页展示所有景点卡片；

2. 景点详情页面：展示景点实拍图、景点名称、门票价格、景点类型、开放时间、游玩路线、收藏数量、地址；提供**购买门票**按钮；下方分标签：景点介绍、游客评论、景区地图；支持收藏景点。

#### 2.2.3 旅游资讯模块

分页展示平台发布的文旅资讯卡片，点击进入资讯详情阅读完整内容。

#### 2.2.4 个人中心模块

左侧功能菜单栏：

1. 个人中心：查看基础账号信息；

2. 修改密码：输入原密码、新密码、确认密码，提交更新登录密码；

3. 我的收藏管理：查看、取消已收藏景点；

4. 景点信息管理：查看本人购票记录；

5. 旅游攻略管理：新增、编辑、删除个人发布的游玩攻略。

#### 2.2.5 新增旅游攻略页面

富文本编辑表单，字段包含：攻略标题、关联景点名称、上传景点配图、游玩天数、用户账号、用户姓名；底部富文本框编辑攻略详细正文，支持文字排版、图片插入。

### 2\.3 管理员后台管理模块（左侧侧边菜单栏）

#### 2.3.1. 首页

后台欢迎首页，展示系统欢迎文字，作为后台功能总入口。

#### 2.3.2. 旅游攻略管理

表格展示所有游客发布的攻略数据，支持检索、查看详情、删除违规攻略，统一管理平台攻略内容。

#### 2.3.3 景点信息管理

包含二级子菜单：

1. **景点类型**：维护景点分类（自然风光、人文古迹等），新增、修改、删除分类；

## 四、核心业务与安全流程

### 4\.1 JWT 双角色权限鉴权流程

1. 游客登录：仅可访问前台页面，仅能操作个人收藏、攻略、购票，无后台系统访问权限；

2. 管理员登录：进入后台管理系统，拥有景点、用户、资讯、攻略全量增删改查权限；

3. 后端拦截器解析 Token 内角色标识，越权访问接口直接返回 403 禁止访问；未携带有效 Token 强制跳转对应登录页面。

### 4\.2 景点浏览 \+ 购票完整业务流程

1. 游客前台浏览景点列表，点击进入景点详情查看图文、价格、游客评价；

2. 点击购买门票生成购票订单，存入订单数据表；

3. 后台管理员可查看全部购票订单，统计各景点门票销量。

### 4\.3 旅游攻略发布与管理流程

1. 游客登录个人中心，进入攻略管理页面，填写标题、配图、游玩天数、富文本正文提交发布；

2. 后台管理员可查看平台所有攻略，删除违规、低质量攻略内容；

3. 所有游客可在前台浏览对应景点关联的游玩攻略。

### 4\.4 景点收藏业务流程

游客在景点详情页点击收藏按钮，系统生成收藏记录；个人中心我的收藏页面统一展示所有收藏景点，支持取消收藏。

## 五、项目部署说明

### 5\.1 SpringBoot 后端部署步骤

1. 执行项目 SQL 初始化脚本，批量创建全部数据表，预置管理员测试账号；

2. 修改`application.yml`核心配置文件：

    - MySQL 数据库连接地址、数据库账号、密码；

    - JWT 加密密钥、Token 过期有效时长；

    - 图片 / 视频文件本地上传存储路径、文件大小、格式限制；

    - 后端服务启动端口（默认 8080）；

3. Maven 打包命令：`mvn clean package -DskipTests` 生成可执行 Jar 包；

4. 服务器运行命令：`java -jar travel-system.jar`。

### 5\.2 Vue 前端部署步骤

1. 前端项目根目录执行 `npm install` 安装全部项目依赖；

2. 修改 Axios 全局请求基地址，指向后端服务接口地址；

3. 生产环境打包静态资源：`npm run build` 输出 dist 文件夹；

4. Nginx 托管 dist 静态页面，配置 /api 反向代理转发至后端 8080 端口；

5. 路由区分游客前台、管理员后台两套独立访问入口。

## 六、系统特色与创新亮点

### 6\.1 业务功能亮点

1. **景点可视化展示体系**：前台首页卡片、详情大图、视频预览多层级展示景点图文信息，游客直观了解景区；

2. **线上攻略分享社区**：游客自主发布游玩图文攻略，形成景区 UGC 内容，丰富平台文旅信息；

3. **一站式线上购票**：景点详情直接下单购票，后台统一管理所有订单，简化线下售票工作量；

4. **多层级景点分类管理**：后台可视化维护景点类型，快速新增自然风光、人文古迹等分类，无需修改代码；

5. **资讯 \+ 景点联动运营**：管理员发布文旅通知、游玩科普资讯，配合景点展示提升平台内容丰富度；

6. **个人中心一站式管理**：游客统一管理密码、收藏、购票记录、个人发布攻略，操作集中便捷。

### 6\.2 系统安全亮点

1. 用户密码采用 MD5 不可逆加密存入数据库，杜绝明文泄露游客隐私；

2. JWT 无状态登录会话，设置固定过期时长，退出登录销毁前端 Token，防止账号劫持盗用；

3. 三层参数校验：前端表单校验 \+ 后端接口参数校验 \+ 数据库字段约束，拦截非法脏数据；

4. 严格角色接口拦截，游客无法访问后台管理接口，管理员账号单独权限隔离；

5. 文件上传安全校验：限制图片、视频后缀格式与单文件大小，过滤恶意脚本上传风险。

### 6\.3 拓展迭代优势

1. 前后端完全解耦分层架构，便于后续拓展在线支付、景区导航地图、景点评分、短信通知、游客留言模块；

2. 基础数据字典化管理：景点类型后台可视化维护，新增分类无需改动后端代码；

3. 全列表多条件检索功能：景点、攻略、订单页面支持关键词搜索，快速定位目标数据；

4. 前台页面轻量化设计，电脑端适配优秀，操作逻辑简单，中老年游客也可快速上手；

5. 全量数据永久归档：景点访问记录、购票订单、攻略、评论全部留存，支持年度文旅数据统计复盘。

## 七、项目总结

本《基于 SpringBoot\+Vue 旅游管理系统》面向文旅景区、线上旅游宣传场景开发，采用前后端分离架构，解决传统线下文旅宣传信息滞后、无线上购票渠道、游客无法分享游玩攻略、景区数据难以统一管理等痛点。系统划分游客、管理员两类核心角色，完整实现景点线上展示、门票线上购买、旅游攻略发布、文旅资讯推送、后台全数据管控一体化功能；配套完善的 JWT 双角色鉴权、密码加密、接口权限拦截安全机制，保障游客账号与平台文旅数据安全。
